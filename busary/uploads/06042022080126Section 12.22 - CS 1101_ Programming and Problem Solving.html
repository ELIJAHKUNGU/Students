12.22 PA06-B: Resizable ASCII Art (15 pts)

Points and submission

This exercise is worth 15 points. You have 8 submission attempts, so use them wisely and try to pass all test cases before running out of submissions. Note also that you have to wait for at least one minute before you can submit again.

Keep in mind that the score you get from zyLabs tells us how many test cases passed and hence, is not your final score for this exercise. The TA will take this into account and will manually look at your submission. The TA will also be looking for incorrect style and other possible deductions before entering your final score on codePost. It is possible that you don't get the full points for the exercise even if you passed all the test cases.

Goals

The overall goals of this exercise is to:

  • Introduce you to nested for-loops.
  • Give you more practice with methods and decision statements.

Problem description

Write a program called ResizableASCIIArt that will printout resizable ASCII Art. The program should ask the user for a size value, then print out the ASCII Art. You should exactly reproduce the format of the output given in the examples. This includes having identical characters and spacing.

Sample run

Input:

4

Run:

Enter a size: 4

       /**\
      //**\\
     ///**\\\
    ////**\\\\
   /////**\\\\\
  //////**\\\\\\
 ///////**\\\\\\\
+*=*=*=*=*=*=*=*=+
|\/\/\/\/\/\/\/\/|
|.\/\/\/..\/\/\/.|
|..\/\/....\/\/..|
|...\/......\/...|
|.../\....../\...|
|../\/\..../\/\..|
|./\/\/\../\/\/\.|
|/\/\/\/\/\/\/\/\|
+*=*=*=*=*=*=*=*=+

You must use the process outlined in class to generate the resizable ASCII art. You cannot just write print statements that prints each line of the figure. In lines that have repeated patterns of characters that vary in number from line to line, represent the lines and character patterns using for-loops. Write pseudo-code and generate tables to understand the patterns as described in class (i.e., the topic of drawing resizable ASCII art).

Another significant component of this exercise is the task of generalizing the program using a single size variable that can be changed to adjust the size of the figure. The code to ask for the size from the user is included in the template code. The value entered by the user is stored in the size variable. You can pass this variable to the different functions that you will write to print out the figure. Note that this is different from the way the examples in class were given, where a global constant was used. In order for us to be able to test your solution, you will instead need to get the input from the user (the size of the ASCII art) and pass that value to each of your functions. You may assume that the value entered by the user will be an integer and the value will be three or greater.

As demonstrated in class, consider coding a solution for one fixed size first. After this is done, look at the versions in the other sizes and observe the differences in terms of the number of instances of the character patterns and the number of lines. Try to find a relationship between these and the size value associated to each drawing.

Avoid significant redundancy; use methods so that no substantial groups of identical statements appear in your code. Also, no print statements should appear in the main method. However, you do not need to use methods to capture redundancy in partial lines, such as the two groups of \/ characters in the line: |.\/\/..\/\/.|

Required program decomposition

For this exercise, the size of any methods in terms of the number of lines used will be restricted. A test case will check that each method you implemented (including the main method) contains at most 20 lines of code. Empty lines, comment lines, and lines containing only an ending curly bracket are not counted.

Our solution decomposes the figure into four parts. Each part is printed by a separate method.

Notes

  • Your solution should work correctly on any positive size value. It is highly likely if you pass the available tests, your solution works correctly for other size values.

  • You can view the output of the ASCII art based on different size values at the following links: [ size 1 | size 3 | size 4 | size 5 | size 8 | size 10 ].

  • Each ASCII art output above represents one instance of the program executing. If we wanted the program to print the figure based on a different size, then the program must be re-run again.

Tips

  • Get one size working first reiterating what was already described above, consider coding a solution for one fixed size first. After this is done, look at the versions in the other sizes and observe the differences in terms of the number of instances of the character patterns and the number of lines. Try to find a relationship between these and the size value associated to each drawing.
